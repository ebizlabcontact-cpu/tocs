## Deploy notes

1. Copy `.env.example` → environment-specific file (never commit secrets).
2. Set `DATABASE_URL`, `NODE_ENV`, `PORT`, `LOG_LEVEL` per `docs/operations/ENVIRONMENT.md`.
3. `loadEnvironment()` runs on `createServer()` / `start()` — invalid config fails fast.
4. Health: `GET /api/v1/health` — liveness without DB.
5. Logs: JSON lines; `LOG_LEVEL=info` recommended for production.
